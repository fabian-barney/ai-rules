## Overview

Release management encompasses the entire process of preparing, documenting, and
publishing a new version of a project. This includes:

1. **Version Planning** - Determining the next version number (semantic versioning)
2. **Changelog Generation** - Documenting changes since the last release
3. **Security Assessment** - Evaluating and documenting security posture
4. **Quality Assurance** - Ensuring all tests pass and quality gates are met
5. **Documentation Updates** - Updating user-facing and technical documentation
6. **Release Artifacts** - Building and publishing release artifacts
7. **Communication** - Announcing the release to stakeholders

## Quick Start

**New to release management?** Start with [QUICK_REFERENCE.md](QUICK_REFERENCE.md)
for a practical overview of the release workflow.

When preparing a release:

1. Review [RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md) for the complete workflow
2. Generate changelog using rules from [CHANGELOG_RULES.md](CHANGELOG_RULES.md)
3. Run security assessment following [SECURITY_ASSESSMENT.md](SECURITY_ASSESSMENT.md)
4. Follow version management guidelines from [VERSION_MANAGEMENT.md](VERSION_MANAGEMENT.md)
